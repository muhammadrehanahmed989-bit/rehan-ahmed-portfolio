 6. HOW THE WORK SECTION RENDERS MEDIA
------------------------------------------------------------------------
  Inside src/pages/Index.tsx there is a `projects` array. Each
  project entry has:
      title  → project name shown on the card
      tag    → small label at the top-left of the media
      desc   → one-line description under the title
      color  → accent glow color for the card
      media  → CDN URL from the imported .asset.json
      type   → "image" or "video"

  The Work component maps this array to cards. If type is
  "image" it renders <img>; if "video" it renders <video>
  with autoPlay + muted + loop + playsInline (so it silently
  plays on all devices, including mobile).

 7. CONTACT FORM  → DELIVERED TO GMAIL
------------------------------------------------------------------------
  The contact form (Contact component in src/pages/Index.tsx) is
  fully functional and delivers every submission straight to:

      muhammadrehanahmed989@gmail.com

  HOW IT WORKS:
    • On submit, the form POSTs a JSON payload (name, email,
      project, message) to FormSubmit's AJAX endpoint:
          https://formsubmit.co/ajax/muhammadrehanahmed989@gmail.com
    • FormSubmit forwards the payload as a nicely formatted
      email to Rehan's Gmail inbox.
    • No backend, no API keys, no database — 100% serverless.
    • A hidden "_honey" honeypot field blocks bots.
    • On success a toast shows "Message sent ✨".
    • On failure a toast asks the user to email directly.

  FIRST-TIME ACTIVATION (one-time):
    The very first time the form is submitted, FormSubmit sends
    Rehan a confirmation email. He must click the link inside
    once — after that, all future submissions arrive automatically
    in his Gmail inbox with no further setup.

  FEEDBACK FORM (Testimonials section → LeaveFeedback):
    Uses the same FormSubmit endpoint. Every submission is
    emailed to Rehan for approval before it is manually added
    to the `testimonials` array. Fields: name, country, role,
    rating (1–5 stars), quote.

  WORK CASE STUDY MODAL (Work section → CaseStudyModal):
    Clicking any project card's arrow (or the media itself)
    opens a liquid-glass modal with client, role, timeline,
    challenge, solution and result bullets. Closes on ESC,
    backdrop click, or the × button. Same behavior on laptop
    and mobile.

  CERTIFICATES CAROUSEL (Certificates section):
    Auto cross-fades through 4 certificates every 4.5 seconds
    using AnimatePresence. Dots and chip buttons let the user
    jump to any specific certificate.


------------------------------------------------------------------------
 8. STYLING CONVENTIONS
------------------------------------------------------------------------
  • Never use raw color classes like text-white or bg-black —
    always use semantic tokens (text-foreground, bg-primary, etc).
  • Glass surfaces: apply `glass` or `glass-strong` +
    `glass-border-glow` for the animated gradient border.
  • Headings use the `font-display` class (Fraunces serif).
  • Body text uses Inter (default).
  • Monospace labels use `font-mono` (JetBrains Mono).
  • Animations:
        Floaty component  → subtle idle up/down floating
        animate-glow-pulse→ pulsing brand glow
        orbit CSS keyframe→ circular revolving motion
        ring-spin(-rev)   → slow rotating rings
